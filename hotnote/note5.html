nat123内网穿透4CentOS7.x

 


 



一，查看操作系统版本。


[root@zkyml ~]# uname -r  #uname -a，或cat /proc/version


3.1 #对应centos7.x




二，系统2.8.x（包括2.8.0）及以上Linux平台安装启动过程（相对简单/推荐）：




1，本地环境更新


[root@zkyml ~]# yum -y update


2，运行环境安装


[root@zkyml ~]# yum install mono-complete


如果提示不存在,请执行


?[root@zkyml ~]# yum install yum-utils


?[root@zkyml ~]# rpm --import "http://keyserver.ubuntu.com/pks/lookup?op=get&search=0x3FA7E0328081BFF6A14DA29AA6A19B38D3D831EF" yum-config-manager --add-repo http://download.mono-project.com/repo/centos/

[root@zkyml ~]# yum makecache




[root@zkyml ~]#yum -y install mono-complete 或者(mono-devel>>monodevelop>>monoDevelop)


#出现Complete完成提示，表示安装成功。


#查看安装运行环境版本信息，注意是大写的V字母


[root@zkyml ~]# mono -V


正常情况下版本号信息是高于2.8.0的。




3，客户端安装


可以直接在自己Linux服务器在线下载安装，也可以登陆网站下载安装包并上传自己Linux服务器。


#将mnt换成是自己本地实际目标安装目录
[root@zkyml ~]# cd /mnt



#下载安装包。过程中可能会有转跳提示，忽略它。


[root@zkyml ~]# wget http://www.nat123.com/down/nat123linux.tar.gz?


#解压本地目录


[root@zkyml ~]# tar -zxvf nat123linux.tar.gz







4，客户端启动

客户端启动及后台服务启动
（1）客户端启动

#可以将mnt换成是自己本地实际安装目录
[root@zkyml ~]# cd /mnt



#运行客户端，并按提示依次输入自己的帐号和密码


[root@zkyml ~]# mono  nat123linux.sh




登录成功正常后，如退出关闭当前进程，需重新以后台服务启动确保nat123的正常运行。

（2）客户端后台服务启动（需要先参与上面的输入帐号密码登录成功记录为前提）

[root@zkyml ~]# cd /mnt     ——将mnt换成是自己本地实际安装目录

[root@zkyml ~]#? mono nat123linux.sh service &  ——后台服务方式启动，自动读取上次成功登录帐号

通过service后台服务方式登录后，该干嘛干嘛的，不影响其他操作，可以ps查看nat123状态




5.?登陆网站远程web管理


登陆网页进行端口映射和动态域名解析添加修改删除，进行管理。用户中心，端口映射列表/域名解析列表相关操作。
