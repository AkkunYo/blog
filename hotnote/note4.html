nat123内网穿透4CentOS6.x

 





一，查看操作系统版本。


[root@zkyml ~]# uname -r #uname -a，或cat /proc/version


2.6   #对应centos6.x


二，系统2.8.x以下Linux平台安装启动过程(仅供参考)




1，本地系统环境更新


[root@zkyml ~]#yum update





2，运行环境mono安装


[root@zkyml ~]#yum  -y install  mono-complete


? #查看安装运行环境版本信息，注意是大写的V字母


[root@zkyml ~]#mono  -V



如果版本号信息是高于2.8.0的，则跳跃到客户端安装启动。


如果小于2.8.0的版本号 或者提示没有mono-complete（使用mono-devel或monoDevelop或monodevelop都没有）则可以手动下载安装mono环境。






环境部署之安装bison环境。如提示已存在时，忽略；如提示是否继续时输入y回车继续。


[root@zkyml ~]#yum  -y install bison gettext





#mnt换成自己（客户端）实际目录


[root@zkyml ~]#cd  /mnt



#在线下载mono2.8或更高版本安装包（部份环境不一定支持最新版本的安装包）


#更多安装可以直接访问http地址查看，省略最后的文件名，保留前台的http目录即可浏览器访问查看。


[root@zkyml ~]#wget  http://origin-download.mono-project.com/sources/mono/mono-2.8.1.tar.bz2


[root@zkyml ~]#wget  http://origin-download.mono-project.com/sources/monodevelop/monodevelop-2.8.1.tar.bz2




#解压本地


[root@zkyml ~]#tar  jvxf  mono-2.8.1.tar.bz2???


[root@zkyml ~]#tar  jvxf  monodevelop-2.8.1.tar.bz2???


#进入mono-2.8.1的解压目录，mnt换成自己的实际目录


[root@zkyml ~]#cd  /mnt/mono-2.8.1 ?






#安装mono环境,——缺什么装什么在运行，直到无报错提示


[root@zkyml ~]#./configure --prefix=/usr



[root@zkyml ~]#make


[root@zkyml ~]#make  install




#安装monodev环境


#进入monodevelop-2.8.1的解压目录


[root@zkyml ~]#cd  /mnt/monodevelop-2.8.1



[root@zkyml ~]#./configure --prefix=/usr



[root@zkyml ~]#make


[root@zkyml ~]#make  install


#安装好mono和monodev后，查看当前mono版本信息


[root@zkyml ~]#mono -V



#确保版本信息提示为2.8.0或大于这个版本号。
